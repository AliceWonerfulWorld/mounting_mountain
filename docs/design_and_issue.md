# ランディングページ デザイン＆実装計画

このドキュメントは、「マウンティングマウンテン」のトップページ（ランディングページ）のデザインコンセプトと実装の詳細をまとめたものです。初心者メンバーの方でも分かりやすいように作成しています。

## 🎨 デザインコンセプト：「頂点を目指せ (Climb to the Top)」

### コアテーマ
- **高さ（Verticality）**: 標高と上昇を強調する（エベレストは8848m）。
- **コントラスト**: 「地上（フッター/開始地点）」と「頂上（ヘッダー/ヒーロー）」の明確な対比。
- **遊び心**: 「マウントを取る」という少し意地悪なテーマなので、エッジが効きつつも親しみやすいデザインに。

### カラーパレット（Tailwind CSS 推奨）
- **メイン**: `bg-zinc-900` / `text-white`（プレミアム感を出すためのダークモード基調）。
- **アクセント**: `text-blue-500` または `text-indigo-400`（冷たく澄んだ高山の空気をイメージ）。
- **アクション**: `bg-red-600`（「マウント」の警告色、またはメインボタン）。

### レイアウト構成
1.  **ヘッダー**: シンプルなロゴ（テキスト）とGitHubへのリンク。
2.  **ヒーローセクション**:
    *   キャッチコピー: 「その発言、標高何メートル？」
    *   ビジュアル: CSSや画像で作った大きな山のシルエット。
    *   **メインボタン**: 「ソロモードで登頂開始」
3.  **特徴紹介**: 「解析」「スコア」「可視化」の3つの特徴をカラムで紹介。
4.  **フッター**: 著作権表示とクレジット。

## 📝 GitHub Issue 下書き

以下は、担当メンバーに渡すIssueのテンプレートです。そのままコピーして使ってください。

### タイトル
`feat: ランディングページ（トップページ）のUI実装`

### 概要

現在 `src/app/page.tsx` はただのリンクがあるだけの簡素な状態です。
ゲームの顔となる、かっこいいランディングページの実装をお願いします！
コンセプトは「頂点を目指せ」です。

### 要件

#### 1. ヒーローセクション（ファーストビュー）
   - **背景**: うっすらとしたグラデーション（例: `bg-gradient-to-b from-zinc-900 to-black`）。
   - **タイトル**: 大きく太いフォントで（例: `text-6xl font-black`）。
   - **サブタイトル**: 「『マウント』を“標高”で可視化するAIゲーム」。
   - **アクションボタン**: `/solo` に遷移する大きなボタン。
     - スタイル: 角丸、高コントラスト（青や赤の背景に白文字）。
     - ホバー効果: カーソルを乗せたら少し大きくなる（`hover:scale-105 transition-transform`）。

#### 2. 特徴紹介セクション（練習として）
   - グリッドレイアウト（`grid-cols-1 md:grid-cols-3`）を使って、以下の3つを並べてください：
     1. **AI解析**: 「あなたの何気ない一言をAIが判定。」
     2. **スコアリング**: 「マウント度が高いほど、標高アップ。」
     3. **ソロトレーニング**: 「まずはソロモードで耐性をつけよう。」

#### 3. 技術的な要件
   - [ ] スタイリングは全て **Tailwind CSS** を使用すること。
   - [ ] スマホでも崩れないようにレスポンシブ対応すること（スマホでは縦並び、PCでは横並び）。
   - [ ] ダークモードに対応すること（`dark:` プレフィックスを活用）。

### 参考リソース
- **アイコン**: [Lucide React](https://lucide.dev/) を使ってください（`npm install lucide-react`済）。
- **コンポーネント**: 必要なら `src/components/ui` にボタンなどの部品を作ってもOKです。

### 完了条件（Acceptance Criteria）
- [ ] ページがかっこよく、ワクワクする見た目になっている。
- [ ] 「ソロモード」ボタンを押すと、正しく `/solo` に移動できる。
- [ ] スマホで見てもレイアウトが崩れていない。
- [ ] リントエラーが出ていない（`npm run lint` が通る）。

## 💡 初心者向け実装のヒント

1.  **まずは構造から**: いきなり色をつけず、まずは `div` や `h1` で骨組みを書いてみましょう。
2.  **配置のコツ**: 真ん中に寄せたいときは、親要素に `flex flex-col items-center justify-center` をつけるのが鉄板です。
3.  **余白が大事**: `gap-4`, `p-8`, `my-12` などを使って、要素同士の間隔を広めに取りましょう。詰まっていると安っぽく見えます。
4.  **真似しよう**: `src/app/solo/page.tsx` のコードなどが参考になるはずです。
